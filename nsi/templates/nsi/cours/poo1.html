Introduction

En première, on a déja vu des stuctures de données concrètes telles que les tableaux indexés (avec le type Python list. Cette année nous allons distinguer les structures de données abstraites (ou types de données abstraits) de leur implémentation à l'aide d'un langage de programmation et de structure de données concrètes.

Une structure de données est une manière d’organiser et de stocker l’information, dont l’objectif est de les traiter plus facilement. Une structure de données doit avoir une interface : un ensemble d'opérations pour ajouter, effacer, accéder, réorganiser, etc… les données.

Type de données abstrait

De façon plus précise, un type de données abstrait (TDA en abrégé) est une spécification :

  • d'un ensemble de données;
  • de l'ensemble des opérations (l'interface) que l'on peut effectuer sur elles

On peut aussi rajouter les préconditions qui définissent les conditions de validité applicables aux opérations.

On peut donc voir un TDA comme un cahier des charges que la structure de données concrètes (l'implémentation) doit respecter. On peut, par ailleurs, avoir plusieurs implémentations possibles du même TDA.

Un exemple : le TDA date

L'interface

On va définir un type de données abstrait appelé date correpondant à l'ensemble des dates écrites sous la forme jj/mm/aaaa. On souhaite effectuer les opérations suivantes sur les dates :

  • créer une date;
  • accéder au jour, au mois et à l'année d'une date;
  • modifier le jour, le mois ou l'année d'une date;
  • comparer des dates;
  • calculer le nombre de jours entre deux dates.

Pour accomplir ces taches, on spécifie l'interface suivante :

  • date(j,m,a) : renvoie un élément de type date créé à partir de trois entiers j (le jour), m (le mois) et a (l'année);
  • getJour(d) : renvoie un entier correspondant jour d'une date d;
  • getMois(d) : renvoie un entier correspondant au mois d'une date d (renvoie un entier);
  • getAnnee(d) : accède à l'année d'une date d (renvoie un entier);
  • setJour(d,j) : modifie le jour de la date d avec l'entier j;
  • setMois(d,m) : modifie le mois de la date d avec l'entier m;
  • setAnnee(d,a) : modifie l'année de la date d avec l'entier a;
  • dateAvant(d1,d2) : renvoie True si la date d1 est avant d2;
  • dateEgal(d1,d2) : renvoie True si les dates d1 et d2 sont identiques;
  • inter(d1,d2) : renvoie un entier correspondant au nombre de jours entre la date d1 et la date d2.

On ajoute à cela une opération permettant d'afficher une date sous la forme d'une chaîne de caractères :

  • afficher(d) : affiche la date sous la forme d'une chaîne de caractères 'jj/mm/aaaa'

Implémentations

Pour implémenter un TDA, c'est-à-dire programmer les opérations sur la structure, il est nécessaire de s'appuyer sur les types de données déjà définis dans le langage utilisé. En Python, on peut avantageusement utiliser les types int, float, boolean ainsi que les p-uplets (avec le type tuple), les tableaux (avec le type list), les dictionnaires (avec le type dict).

On va ici envisager deux implémentations, l'une avec un tableau et l'autre avec un dictionnaire. Pour faire court, on passera sous silence les préconditions. Par exemple un entier représentant la valeur d'un mois doit être compris entre 1 et 12.

Conclusion

L'interface apporte toutes les informations nécessaires pour utiliser le type de données abstrait. Ainsi, le programmeur qui s'en sert n'a pas à se soucier de la façon dont les données sont représentées ni de la manière dont les opérations sont programmées (il en fait abstraction). L'interface (uniquement) lui permet d'écrire toutes les instructions qu'il souhaite et obtenir des résultats corrects. Par exemple, dans le cas précédent, l'implémentation du type date n'est pas la même dans les deux cas (en particulier la fonction dateAvant). Pourtant, lorsque les opérations sont appliquées aux mêmes données, on obtient les mêmes résultats.
